# AETHER 拟物化 AI 智能体应用 - 产品功能概要设计文档

## 1. 产品定位与核心理念

**AETHER** 是一款基于前端架构构建的**单页功能性agentic应用**。应用完全舍弃了传统的菜单导航模式（**无菜单模式**），功能界面采用**功能服务卡片居中**的方式，确保用户在单一场景内的深度聚焦。

产品的核心灵魂是 **Airobot（内嵌功能型智能体 AETHER）**。它不仅仅是一个被动的文字对话框，而是一个具有情境感知（上下文理解）、主动触发（闹钟唤醒、整点报时）、多模态交互并在其基础上**基于用户功能需求实现功能卡片的自主服务**的效率助理。

系统的基础设定（核心约束）如下：
*   **单页功能性agentic应用，无菜单模式，功能服务卡片居中**
*   **Airobot无边框拟物、卡通动画风格，跟随功能卡片切换状态**
*   **功能服务采用单一职责的功能卡片，提供按需功能卡片服务**
*   **静态的主功能服务卡片，采用拟物化、隐喻式功能旋钮切换**
*   **服务型功能卡片，采用动态、隐喻化设计的弹出式拟物卡片**
*   **内嵌功能型智能体，基于用户功能需求实现卡片的自主服务**
*   **clear模块化架构，airobot与功能分离，UI要组件化，模块化**

## 2. 功能模块划分与设计

项目整体上按业务垂直划分为四大功能模块以及一个支撑全局的底层智能服务引擎模块。

### 2.1 Airobot (核心 AI 智能体模块)
定位产品的大脑与交互界面，融合拟人物理反馈动效，游离于固定功能之外但又总览全局。

*   **拟物化虚拟形象**: 位于界面显著位置的机器人 UI 动画，随业务状态（如说话、倾听、思考、沉睡、闪烁）随时改变自身形象和动作反馈。
*   **多模态对话交互**: 
    *   **常规对话**: 支持文字向 AI 发起对话，AI 根据上下文输出活泼、拟人的文字回应并同时触发语音播报（TTS）。
    *   **Live 语音对讲**: 深度集成实时语音流控制（Gemini Live API），允许用户通过长按或触发功能进入“实时聆听与口语交流”状态。
*   **主动式助理服务**:
    *   能够从环境和子模块接入事件并主动发声，如通过时钟触发的播报、闹钟唤醒时的事件催促与早间简报。
    *   提供“伴随式”智能建议扩展（如询问天气、简报今天所有的日程安排）。

### 2.2 时钟与计时模块 (Clock)
兼顾装饰与实用工具属性的沉浸式时间管理功能。

*   **拟物表盘查看**: 提供高精度的模拟指针以及数字与时区切换组合的拟物化界面。
*   **AI 唤醒闹钟**:
    *   闹钟设定支持录入“待办事项/唤醒目的”。
    *   当闹钟触发时，不是单纯的铃声，而是联动 Airobot 生成专属语境的个性化起床催促或事务播报。
*   **全景专注计时 (番茄钟/倒计时)**: 支持时间设定、进度追踪，帮助用户完成专注工作闭环。
*   **整点报时**: 到达大时整点，隐性联动 AI 主动播报时间并附带鼓励性短语。

### 2.3 日程与待办模块 (Schedule)
作为用户事务管理的承载地，强调极简和高效录入。

*   **清单检视 (Todo List)**: 列表化呈现今日及未来的核心待办，打通状态切换（未完成/已完成）。
*   **全景日历视图**: 基于日历卡片的形式鸟瞰一个月的事务分布，提供快速的日程跳读。
*   **日程计划管理**: 支持基础的增删改查动作，供 Airobot 随时获取后台数据做“日程简报”使用，支持与语音指令录入待办（未来持续拓展）。

### 2.4 智能播客模块 (Podcast)
基于 AIGC 即时生成的富文本内容消费功能，打破传统知识获取方式。

*   **话题订阅与即时制作**: 用户可快速选择内置主题或与 Airobot 交互给定概念，AI 将为其撰写并生成完整的“故事/科普文章”。
*   **拟物化播放器**: 提供磁带机/黑胶唱片质感的播放控制台。
*   **AI 语音演绎**: 依赖 TTS 技术将生成的长文本合成有温度的长音频（Podcast 剧集），提供连续播放、暂停、进度调整能力。

## 3. 系统底层支撑机制 

### 3.1 Agent Service (智能核心驱动层)
*   **意图与流转**: 承接文本与 Live 语音底层的控制流。将从组件解耦，负责收发大模型双向信令，判定用户指令意图是“闲聊”还是“具体功能调用”（如“帮我定个五分钟后的闹钟”、“帮我新建一个晚上的会议日程”）。
*   **模型统一抽象**: 收拢对大语言模型的各种交互调用（生成内容、生成TTS音频片段）。

### 3.2 交互隐喻与界面架构
*   **FunctionalModulePlate (卡片基座)**: 纯纯粹淬的沉浸式占位容器，每次只载入上述的一个功能模块子树（由单向数据流控制），配合 Framer Motion 呈现流体动画进出。
*   **SkeuomorphicDial (主干旋钮)**: 界面下方常驻机械旋钮组件。通过旋转阻尼动效驱动应用在“时钟”、“日程”、“播客”等状态间安全流转，避免常规网页菜单对沉浸感的破坏。
