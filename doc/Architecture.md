# AETHER 拟物化 AI 智能体应用架构文档

## 1. 项目简介

本项目是一个基于**单页功能性agentic应用**理念构建的沉浸式 AI 个人助理工具。应用完全舍弃了传统的菜单导航模式（**无菜单模式**），功能界面采用**功能服务卡片居中**的设计，保障用户的纯粹聚焦体验。

项目的核心理念与约束如下：
*   **单页功能性agentic应用，无菜单模式，功能服务卡片居中**。
*   **Airobot无边框拟物、卡通动画风格，跟随功能卡片切换状态**。
*   **功能服务采用单一职责的功能卡片，提供按需功能卡片服务**。
*   **静态的主功能服务卡片，采用拟物化、隐喻式功能旋钮切换**。
*   **服务型功能卡片，采用动态、隐喻化设计的弹出式拟物卡片**。
*   **内嵌功能型智能体，基于用户功能需求实现卡片的自主服务**。
*   **clear模块化架构，airobot与功能分离，UI要组件化，模块化**。

## 2. 核心功能模块

项目采用 CLEAR 模块化架构（清晰、松耦合），智能体（Airobot）与各功能模块高度解耦。主要包括四大核心功能：

1. **Airobot (AI 智能体大脑与交互)**
   - **特性**: 结合拟人化、卡通动画风格，无缝贯穿各个功能。
   - **核心能力**: 全局语音对话、语音响应、实时打断监听、主动作动（如整点唤醒语音播报时间与天气/日程简报，闹钟响时督促起床与播报提醒规则）。

2. **Clock (时钟与计时模块)**
   - **特性**: 拟物表盘设计。
   - **核心能力**: 
     - 基础时钟查看。
     - AI 闹钟设置与触发（与 Airobot 桥接）。
     - 番茄钟/专注计时。

3. **Schedule (日程与待办)**
   - **特性**: 卡片化平滑呈现，沉浸级检视体验。
   - **核心能力**: 当日待办清单（Todo）、全景日历试图、行程计划增删改查。

4. **Podcast (AI智能播客)**
   - **特性**: 基于生成式 AI 的长音频内容消费。
   - **核心能力**: 根据话题自动生成资讯、故事或知识播客；音频合成与拟物化播放器控制。

5. **Services (Agent 大脑服务层)**
   - **特性**: 系统基础设施抽象层，对大模型接口的重度封装。
   - **核心能力**: 目前已具有基础的 `geminiService`（含部分文本、TTS调用），但为增强 Airobot 对整个单页各模块场景的**认知与决策**，正将其升级与抽离出一层专属的 `Agent Service`：借以封装意图识别、上下文路由、复杂功能模块调用（Tools-call）以及更底层的 Live 语音控制流接入，彻底与视图层解耦。

## 3. 技术选型与技术栈

- **前端框架**: React 19 + TypeScript
- **构建工具**: Vite
- **UI & 动效**: Tailwind CSS 4 + `motion/react` (提供丰富、流畅的卡片与拟物组件切换动效)
- **AI 驱动**: `@google/genai` (搭载 Gemini 核心大语言能力，包含文本对话与 TTS 语音合成流)
- **状态与逻辑管理**: 基于 React Custom Hooks 的模块化状态切片分离（`useAether`, `useClock`, `useSchedule`, `usePodcast`）。

### 4.2 数据级与交互级架构图

Airobot（智能体服务）游离于业务之上又赋能各模块。`App.tsx` 充当**中央神经节点**，将松耦合的功能模块通过 Props 和 Callback 绑定给 Airobot 进行事件传递。

```text
+-----------------------------------------------------------+
|                        App.tsx                            |
|  (中央调度核心 / 状态汇集 / 界面大盘 / 拟物切换枢纽)        |
+---+----------------------+-----------------------+--------+
    |                      |                       |
    | [Props & Callbacks]  |                       |
    v                      v                       v
+--------+            +---------+             +----------+
|  Clock | <--------> | airobot | <---------> | Schedule |
| Module | (整点/闹钟)| (Aether)| (代办询问等)|  Module  |
+--------+    触发      +----+----+    触发       +----------+
                             |
                             | (播客制作)
                             v
                        +---------+
                        | Podcast |
                        | Module  |
                        +---------+

* 交互架构：
  [View 层] -----无边框拟物UI / Framer Motion ----> [用户空间]
  [控制器] ------Custom Hooks 解耦提取业务------> [React State]
  [AI Agent] ---@google/genai 桥接---------------> [Gemini 核心]
```

## 5. 设计理念落地

- **模块化**: 通过 `useXXX.ts` (如 `useClock`, `useAether`) 抽离了业务侧数据。做到组件化（UI）与模块化（Logic）严格解耦。
- **界面体验**: 组件遵循单一职责，保证单功能单页聚焦。使用 `FunctionalModulePlate` 和 `SkeuomorphicDial` (功能旋钮) 达成无菜单、通过隐喻旋钮交互切换模块的要求。
- **智能化互通**: Airobot 不再是孤立的聊天框。时钟模块发现到达整点或触发闹钟时，会主动调用 `baseHandleRobotChat()` 注入上下文与系统级 Prompts，做到 AI 基于环境状态（时间、日程、设定）的主动拟人化反馈。
